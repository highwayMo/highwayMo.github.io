final修饰符

为什么使用final修饰符？

继承关系最大的弊端是破坏封装:子类可以访问父类的实现细节，而且可以通过方法覆盖的形式修改实现细节。

  1. 被final修饰的类：该类不能再拥有子类
    只要满足以下条件就可以用final修饰

    • 某类不是专门为继承设计的
    • 安全考虑，不想被修改
    • 确信该类不会再被扩展

  2. 被final修饰的方法:该方法不能再被覆盖
    只要满足以下条件就可以用final修

    • 父类中提供的算法骨架，子类不能覆盖
    • 在构造器中调用的方法**(初始化方法)**,此时一般用final修饰。

  3. 被final修饰的变量:该变量不能再被赋值(即常量)

    • final变量必须初始化，系统不会给final字段初始化

    • 全局静态常量:public static final修饰的变量直接使用类名调用，类似于C语言的define

      被final修饰的变量类型 结果
      引用类型变量 该变量参考的地址不变，但是参考地址里的内容能被改变
      基本类型变量 该变量变为常量，不能再被赋值

final是唯一可以修饰局部变量的修饰符
只要满足以下条件就可以用final修饰
- 在程序中，多个地方需要使用到的常量EX:PI
- 在开发中，一般会专门写一个常量类供其他类来使用
