C是怎么变成汇编的

这一段其实不是很多，就是加深了对c的理解而已

裸函数__declspec(naked)

怎么样对C变成汇编产生最直观的理解呢？那必须非裸函数莫属了。普通的函数通过反汇编会看到编译器自动生成的多行汇编代码。如果加上裸函数的声明，编译器就不会做任何添加。下面写一个裸函数来进行对比。

普通函数

int function(int x,int y,int z){
    int a=4;
    int b=5;
    int c=6;
    return x+y+z+a+b+c;
}

用汇编来实现是

int __declspec(naked) function(int x,int y,int z)
{
	__asm
	{
		//提升堆栈
		push ebp
		mov ebp,esp
		sub esp,0x40
		//保留现场
		push ebx
		push esi
		push edi
		//填充缓冲区
		mov eax,0xCCCCCCCC
		mov ecx,0x10
		lea edi,dword ptr ds:[ebp-0x40]
		rep stosd
		//增加局部变量
		mov dword ptr ds:[ebp-0x4],0x4
		mov dword ptr ds:[ebp-0x8],0x5
		mov dword ptr ds:[ebp-0xc],0x6
		//实现函数功能
		mov eax,dword ptr ds:[ebp+0x8]
		add eax,dword ptr ds:[ebp+0xc]
		add eax,dword ptr ds:[ebp+0x10]
		add eax,dword ptr ds:[ebp-0x4]
		add eax,dword ptr ds:[ebp-0x8]
		add eax,dword ptr ds:[ebp-0xc]
		//恢复现场
		pop edi
		pop esi
		pop ebx
		//降低堆栈
		mov esp,ebp
		pop ebp
		//
		ret	
	}
}

短短的几行代码，实际上在底层其实非常的复杂。因为窝是初学者在写的时候犯了几个错误

  • rep的计数器是ECX
  • 取压入堆栈的参数至少也是$[ebp+0x8]$开始,因为用call的时候会把下一行指令的地址压栈。(目前窝的水平只有这样的情况)
